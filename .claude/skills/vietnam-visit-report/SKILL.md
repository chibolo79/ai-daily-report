## 이 스킬이 하는 일

**역할 1 — 방문 전 브리핑** (트리거: "방문 전", "미팅 준비", "오늘 방문할 업체", "사전 리뷰"):
1. 분석 파일에서 해당 업체 정보 읽어 브리핑 출력
2. 업체별 방문 체크리스트 채팅창 출력

**역할 2 — 공식 보고서 작성** (트리거: "보고서 작성", "출장 보고서"):
1. 방문 기업/기관 현황 구조화
2. 시장 현황 분석
3. 후속 액션 플랜

**출력 형식:**
- 마크다운 (`YYYY-MM-DD_Vietnam_Visit_Report.md`)
- ~~HTML~~ → 채팅창 텍스트로 대체 (토큰 절약)

## 보고서 구조

### 📋 헤더
- 출장 기간, 방문 도시, 작성자, 작성일

### 🏢 파트 A: 방문 기업/기관 현황
각 방문처마다:
- **기업/기관명** + 홈페이지 링크
- **담당자** — 이름, 직함
- **미팅 일시 및 장소**
- **미팅 목적** — 사전 아젠다
- **주요 논의 내용** — 핵심 내용 불릿 정리
- **상대방 반응/온도** — 긍정/중립/부정 + 구체적 코멘트
- **즉시 팔로업 사항** — 보내야 할 자료, 답장해야 할 질문

### 📊 파트 B: 시장 현황 분석
- **시장 규모 & 성장률** — 수치 기반
- **주요 플레이어** — 로컬 경쟁사, 글로벌 경쟁사
- **기회 요인** — 진입 가능성이 높은 영역
- **리스크 요인** — 규제, 현지화 이슈, 경쟁 강도
- **현지 파트너 후보** — 미팅에서 언급된 협력 가능 기업

### ✅ 파트 C: 후속 액션 플랜
표 형식 (우선순위 | 액션 | 담당 | 기한 | 상태):
- 미팅 후 발송 자료
- 다음 미팅 일정 조율
- 내부 공유 사항
- 중장기 검토 과제

## 방문 전 리뷰 요청 시 응답 순서

업체 방문 전 정보를 물어볼 때 아래 순서로 응답한다:

1. **분석 파일 기준 방문 전 리뷰 먼저 출력** — 기본 정보 / 회사 성격 / 세일즈 현황 / 방문 핵심 체크포인트 / 영업 접근 방향
2. **체크리스트 출력** — 공통 질문 C1~C7 + 업체별 특이 질문을 텍스트로 출력 (HTML 파일 생성 금지)
3. **추가 리서치 여부 문의** — 브리핑 마지막에 반드시 아래 문구로 확인:
   > "추가 웹 리서치가 필요하시면 말씀해 주세요. (공식 홈페이지 기준 정보만 파일 반영 가능)"

**웹 조사(보충 리서치) 규칙**:
- 방문 전 리뷰 시 웹 조사는 **하지 않는다**. 분석 파일 기존 내용만 사용.
- 웹 조사는 사용자가 "조사해줘", "찾아봐줘", "리서치해줘" 등을 **명시적으로 요청할 때만** 실행.
- 단, Volza·Panjiva 등 유료 DB 기반 정보는 파일 반영 금지. 채팅에서 "미검증" 명시 후 사용자 판단에 맡긴다.

**정보 신뢰도 규칙 (필수 — 실제 영업 판단에 직결)**:
- **검증된 정보만 분석 파일에 기재한다.** 추정·추론·AI 요약 기반 정보는 기재 금지.
- **웹 조사 결과의 신뢰도 판단 기준**:
  - ✅ 기재 가능: 업체 공식 홈페이지, 공식 보도자료, 사용자가 직접 확인한 정보
  - ✅ 기재 가능: 공개된 통계(직원 수, 설립연도 등) — 출처 명시 조건
  - ❌ 기재 금지: 유료 데이터베이스(Volza, Panjiva 등) AI 요약에서만 나온 거래처·공급처 이름
  - ❌ 기재 금지: 검색 결과 snippet에서 추론된 내용
  - ❌ 기재 금지: "~으로 추정", "~가능성 있음" 수준의 정보를 사실처럼 기재
- **불확실한 정보는 채팅에서도 명확히 구분해서 전달한다**: "확인되지 않은 내용입니다" 선언 후 제시. 분석 파일에는 기재하지 않음.
- **웹 조사 후 출처를 반드시 명시하고, 직접 접근 불가(유료 등)인 경우 즉시 사용자에게 알린다.**

## 방문 체크리스트 공통 질문 (모든 업체 필수)

아래 7개는 업체에 관계없이 매 방문마다 반드시 확인한다.

| # | 질문 | 목적 |
|---|------|------|
| C1 | DSR 제품 중 현재 사용 중인 품목은? | 현 거래 현황 파악 |
| C2 | DSR 포함 전체 사용 제품 중 가장 많이 쓰는 품목·공급처는? | 경쟁 구도 파악 |
| C3 | 전체 wire 수요량 (월/연) | 시장 규모 파악 |
| C4 | 향후 비즈니스 전망 (수요 증감, 신규 프로젝트 여부) | 파이프라인 파악 |
| C5 | 업체의 주요 경쟁사 / DSR의 경쟁사(공급처 기준)는? | 경쟁사 현황 파악 |
| C6 | **서플라이 체인** — DSR wire가 최종적으로 어디에 쓰이는지 (End User까지) | 체인 구조 파악, 품질 기준 예측 |
| C7 | **4M 등록 현황** — DSR wire로 4M 등록되어 있는지, 공급처 변경 절차는? | Lock-in 여부 및 경쟁사 진입 가능성 판단 |

체크리스트 생성 시 공통 질문 7개를 앞에 배치하고, 업체별 특이 질문을 뒤에 추가한다.

## 업체 섹션 작성 규칙 (필수 — 전 업체 공통 적용)

### 섹션 구조 (순서 고정)

```
### [회사명]
| 항목 | 내용 |
|------|------|
| **담당자** | ... |
| **주소** | ... |
| **연락처** | ... |
| **이메일** | ... |
| **홈페이지** | ... |

**회사 개요**: ...
**세일즈 현황**: ...
**품질 이슈 이력**: ...  ← 이슈가 있는 거래처만 기재
**논의사항**: ...        ← 이전 방문 있으면 기존/금번 구분. 첫 방문이면 구분 없이 서술
**FOLLOW UP**: ...
```

### 작성 규칙 상세

| 규칙 | 내용 |
|------|------|
| **영문 표기** | 모든 영문 대문자. 담당자명·직함·품목·지명·인증명 전부 포함 |
| **WIRE 사이즈** | 소수점 둘째 자리 고정. 예: 0.20MM, 2.40MM |
| **범위 표기** | `~` 대신 `-` 사용. 마크다운 취소선 방지 |
| **어투** | `~임/있음/하였음`. `~입니다/있습니다` 금지 |
| **서술 방식** | 결재권자가 배경지식 없이 읽어도 이해되도록 서술형으로 풀어서 작성. 키워드 나열 금지 |
| **논의사항 구조** | 이전 방문 있으면: `기존 논의사항`(이전 방문 통합, 날짜·차수 생략) + `금번 논의사항`. 첫 방문이면 구분 없이 내용만 서술 |
| **부제목** | `방문 이력` → `논의사항` / `영업 포인트` → `FOLLOW UP` |
| **방문 이력 항목** | `미방문` 또는 `첫 방문` 단독 표기 시 해당 항목 삭제. 논의사항 안에 통합 서술 |
| **"차기 방문 시"** | FOLLOW UP에 "차기 방문 시 확인" 단독 문구 금지. 향후 재확인 필요 등으로 표현 |
| **주소 비고(※)** | 주소 혼선·랜드마크 안내 등 내부 비고는 보고서에서 삭제 |
| **미확인 항목** | 공란 불가. `(미확인 — 방문 시 확인 필요)` 명시 |
| **세일즈 현황 항목명** | `수요 품목` → `DSR 수요 품목` / `수요량` → `전체 WIRE 수요량` |

## 성공 기준

✅ **보고서 필수 포함 항목:**
- [ ] 방문처별 미팅 내용 구조화 (담당자, 논의 내용, 팔로업)
- [ ] 상대방 반응/온도 명시
- [ ] 시장 분석 (기회/리스크 균형 있게)
- [ ] 액션 플랜 표 (기한 포함)
- [ ] 마크다운 파일 생성 (HTML 불필요)

❌ **자주 발생하는 실수:**
- 미팅 내용을 일기처럼 나열하는 것 — 구조화된 불릿으로 정리
- 후속 액션 없이 끝내는 것 — 반드시 기한과 담당자 명시
- 상대방 반응 누락 — 온도 파악이 핵심
