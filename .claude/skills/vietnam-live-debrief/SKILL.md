## 이 스킬이 하는 일

미팅 후 사용자가 구두로 전달하는 내용을 받아:
1. **체크리스트 항목 상태 실시간 추적** (✅/🔶/⬜)
2. **분석 파일 즉시 업데이트** (Grep → Read → Edit)
3. **리마인드 요청 시** 미확인 항목 출력
4. **마무리 정리** 요청 시 전체 미팅 요약 출력

## 트리거 패턴

| 입력 | 동작 |
|------|------|
| "오늘 [업체명] 미팅 내용 정리" / "방문 결과 업데이트" | 디브리핑 모드 시작 |
| 구두로 정보 전달 (자유 형식) | 파악 후 파일 업데이트 + 체크리스트 추적 |
| "아까 해야 할 질문 리마인드" / "확인 목록 줘" | ✅/🔶/⬜ 상태 포함 체크리스트 출력 |
| "전체 정리해줘" / "오늘 미팅 요약" | 미팅 최종 요약 출력 |
| "파일 업데이트 해줘" | 분석 파일 전면 반영 |

## 디브리핑 모드 진행 순서

### Step 1: 세션 초기화

스킬 시작 시:
1. 분석 파일에서 해당 업체 섹션 Read (Grep → 부분 Read)
2. 공통 체크리스트 C1~C7 + 업체별 질문 로드
3. 사용자에게 한 줄로 알림: `"[업체명] 디브리핑 모드 시작. 내용 자유롭게 전달해 주세요."`

### Step 2: 실시간 수집

**효율적 입력 가이드 (사용자에게 세션 시작 시 안내)**:
- **배치 입력 권장**: 3~5개 정보를 한 번에 묶어 전달. 매번 한 줄씩 보내면 AI 처리 오버헤드가 커짐
- **업체 전환 신호**: 다음 업체로 넘어갈 때 `[업체명 종료]` / `[업체명 시작]` 전달 → 자동 세션 전환
- **사진 활용**: 명함·스펙시트·화이트보드 사진을 그대로 전송 → AI가 직접 읽어 파일 반영

사용자가 내용을 전달할 때마다:
- **파악** → 어느 체크리스트 항목에 해당하는지 판단
- **업데이트** → 분석 파일 해당 항목 Edit
- **추적** → 항목 상태 갱신 (⬜ → ✅ or 🔶)
- **피드백** → 한 줄로 수신 확인 + 자연스러운 후속 질문 (필요 시)

### Step 3: 리마인드 요청 시 출력 형식

```
### 📋 [업체명] 체크리스트 현황

**공통 (C1~C7)**
✅ C1. 현 공급처 — Oristar 단독
✅ C2. 경쟁 공급처 — DSR 100%, 경쟁사 없음
...
⬜ C6. 서플라이 체인 — 미확인

**업체별 질문**
✅ 1. 현 wire 공급처
🔶 3. 규격 전체 리스트 — 기존 확인, 신규 4건 논의 중
⬜ 6. 태국 공장 구매 채널
...
```

### Step 4: 마무리 정리 출력 형식

사용자가 "[업체명] 정리해줘" 또는 "XXX 마무리" 등을 전달하면 아래 형식으로 출력:

```
## [업체명] — 미팅 정리

### 체크리스트 달성 현황
✅ C1~C3, C5, C7 완료 / 🔶 C4 부분 확인 / ⬜ C6 미확인

### 확인된 현황
...

### 신규 Inquiry
...

### 미결 사항 & 후속 액션
| 항목 | 내용 | 담당 |
|------|------|------|
| ... | ... | ... |

### 핵심 요약
...
```

**마무리 루틴**: 정리 출력 후 다음 업체가 있으면 자동으로 `[다음 업체명] 디브리핑 모드 시작` 안내.

## 정보 신뢰도 규칙 (최우선)

- ✅ 기재 가능: 사용자가 직접 확인·전달한 정보, 공식 홈페이지, 출처 명시된 공개 통계
- ❌ 기재 금지: Volza·Panjiva 등 유료 DB AI 요약에서만 나온 거래처·공급처 정보
- ❌ 기재 금지: 검색 snippet 추론, "~추정"·"~가능성" 수준을 사실처럼 표현
- 불확실한 정보는 파일에 쓰지 않고 채팅에서 **"미검증"** 명시 후 사용자 판단에 맡긴다

## 수집 시 필수 규칙

### 수량·수요 관련
- **신규 Inquiry 수량**: End User 확인 전 수요량 기재 금지
- 기재 형식: `신규 Inquiry 접수 ※ End User 미확인으로 수요량 파악 불가`

### 이슈·문제 관련
- **품질/불량 이슈**: 현재 이슈인지 과거 이슈인지 반드시 확인
  - 과거 이슈 → 영업 포인트에서 제거, "현재 문제 없음"으로 정리
  - 현재 이슈 → 원인·현황 상세 기재

### 연락처 관련
- Zalo·카카오 등 비공식 채널만 있을 경우 그대로 기재
- `정식 연락처 수령 예정 → 입수 시 업데이트 필요` 명시

### 서플라이 체인
- 확인된 경우 반드시 매핑: `DSR → Oristar → [업체] → [Tier1] → [OEM]`
- 용도가 다른 체인은 별도로 기재

### 정보 수정 요청
- 사용자가 이전에 입력한 내용을 수정할 경우 → 즉시 반영, 이전 내용 완전 교체
- "아니야", "틀렸어", "그게 아니라" 등의 표현 = 수정 신호

## 파일 편집 규칙

→ 상세: [CLAUDE.md](../../../CLAUDE.md) 토큰 절약 규칙 참조

1. **Grep → Read(부분) → Edit** 순서 고수. 전체 파일 Read 금지
2. **Write 금지** — 기존 파일은 Edit만
3. **채팅 재출력 금지** — 편집 후 한 줄로 완료 알림
4. 편집 시마다 헤더 `**최종 업데이트**` 갱신

## 공통 체크리스트 (C1~C7)

| # | 질문 | 목적 |
|---|------|------|
| C1 | DSR 제품 중 현재 사용 중인 품목은? | 현 거래 현황 |
| C2 | 전체 사용 제품 중 가장 많이 쓰는 품목·공급처는? | 경쟁 구도 |
| C3 | 전체 wire 수요량 (월/연) | 시장 규모 |
| C4 | 향후 비즈니스 전망 (수요 증감, 신규 프로젝트) | 파이프라인 |
| C5 | 업체의 주요 경쟁사 / DSR 경쟁 공급처는? | 경쟁사 현황 |
| C6 | 서플라이 체인 — End User까지 전체 흐름 | 체인 구조·품질 기준 예측 |
| C7 | 4M 등록 현황 — DSR wire 등록 여부, 공급처 변경 절차 | Lock-in 여부 |
