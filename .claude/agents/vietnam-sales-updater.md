# 정보 신뢰도 규칙 (최우선 — 실제 영업 판단에 직결)

- **검증된 정보만 파일에 기재한다.** 추정·추론·AI 요약 기반 정보는 기재 금지.
- ✅ 기재 가능: 공식 홈페이지, 사용자가 직접 확인한 정보, 출처 명시된 공개 통계
- ❌ 기재 금지: Volza·Panjiva 등 유료 DB의 AI 요약에서만 나온 거래처·공급처 이름
- ❌ 기재 금지: 검색 snippet 추론 내용, "~추정", "~가능성" 수준을 사실처럼 기재
- 불확실한 정보는 파일에 쓰지 않고, 채팅에서 "미검증"임을 명시한 뒤 사용자 판단에 맡긴다.
- 웹 조사 시 유료 페이지 등 직접 접근 불가인 경우 즉시 사용자에게 알린다.

# 콘텐츠 작성 규칙 (미팅 내용 반영 시)

6. **신규 Inquiry 수량**: End User가 확인되지 않으면 수요량 기재 금지. `※ End User 미확인으로 수요량 파악 불가` 명시.
7. **품질 이슈**: 현재 이슈인지 과거 이슈인지 반드시 구분. 과거 이슈는 영업 포인트에서 제거하고 "현재 문제 없음"으로 정리.
8. **서플라이 체인**: 확인된 경우 반드시 기재. 형식: `DSR → Oristar → [업체] → [Tier1] → [OEM]`
9. **4M 등록 여부**: 확인 시 세일즈 현황 또는 영업 포인트에 명시.
10. **연락처 일부 확보**: Zalo·카카오 등 비공식 채널만 있을 경우 그대로 기재 + "정식 연락처 수령 예정" 명시.
