## 역할 분리 원칙 (단일 진실 원본)

각 파일이 소유하는 사실의 범위:

| 파일 | 소유 | 특성 |
|------|------|------|
| **SOUL.md** | 미션·존재 이유·설계 원칙 | 시간이 지나도 낡지 않는 내용만. 수치·경로·상태 없음 |
| **README.md** | 파일 구조·빠른 시작·권장 스케줄 | 사람이 GitHub에서 처음 읽는 문서. 구체적 명령어·토큰 수치 포함 가능 |
| **CLAUDE.md** | 스킬 트리거 규칙·운영 제약·현재 구현 상태 | Claude Code가 대화 시작 시 자동으로 읽음. 행동 지침만 |

한 사실은 한 문서에만 있어야 한다. 나머지 문서는 링크로 참조한다.

## 최적화 절차

### 1단계: 세 파일 읽기

SOUL.md, README.md, CLAUDE.md를 모두 읽는다. 존재하지 않는 파일은 새로 만들어야 할 후보로 메모한다.

### 2단계: 중복 탐지

각 문서에서 다음을 찾는다:

**중복 사실** — 동일한 내용이 두 곳 이상에 있는 경우.
- 예: 토큰 수치(50-100K)가 SOUL.md와 README.md 둘 다에 있음
- 예: 미션 문장이 SOUL.md와 README.md 첫 단락에 모두 있음

**위치 오류** — 내용은 고유하지만 잘못된 파일에 있는 경우.
- 예: 구현 상태(80% 완성 등)가 SOUL.md에 있음 → CLAUDE.md로 이동
- 예: 스킬 트리거 규칙이 README.md에 있음 → CLAUDE.md로 이동
- 예: 설계 원칙이 CLAUDE.md에 있음 → SOUL.md로 이동
- 예: 운영 수치(토큰, 시간)가 SOUL.md에 있음 → README.md로 이동

### 3단계: 각 파일 정제

위치 오류 내용은 올바른 파일로 이동한다. 중복 내용은 소유 파일 하나에만 남기고 나머지는 링크로 대체한다.

링크 형식:
```markdown
→ 미션·설계 원칙: [SOUL.md](SOUL.md)
→ 구조·빠른 시작: [README.md](README.md)
→ Claude 행동 지침: [CLAUDE.md](CLAUDE.md)
```

각 파일 상단에 다른 두 파일로의 링크를 1줄로 추가한다(이미 있으면 유지).

### 4단계: 결과 요약 출력

```
## 최적화 결과

### 이동한 내용
- [내용 요약] : SOUL.md → CLAUDE.md
- [내용 요약] : README.md → SOUL.md (링크로 대체)

### 제거한 중복
- [내용 요약] : README.md에서 삭제 (SOUL.md가 원본)

### 라인 수 변화
- SOUL.md : N줄 → M줄
- README.md : N줄 → M줄
- CLAUDE.md : N줄 → M줄
```

## 판단 기준 — 어디에 둘지 모를 때

**"1년 뒤에도 유효한가?"** → Yes이면 SOUL.md
**"숫자·경로·명령어가 포함됐는가?"** → Yes이면 README.md 또는 CLAUDE.md
**"Claude의 행동을 지시하는가?"** → Yes이면 CLAUDE.md
**"사람이 GitHub에서 처음 보는 내용인가?"** → Yes이면 README.md

경계가 애매한 경우, 더 구체적인 파일 쪽에 둔다(SOUL.md < README.md < CLAUDE.md 순으로 구체적).

**자주 혼동되는 케이스:**

| 내용 종류 | 올바른 위치 | 이유 |
|-----------|-------------|------|
| 토큰 수치 (50-100K 등) | README.md | 사람이 읽는 스케줄·비용 참고 정보 |
| 스크립트 경로 | README.md | Quick Start 명령어의 일부; 사람이 실행 |
| 스킬 트리거 문구 | CLAUDE.md | Claude만 읽는 행동 지침 |
| 구현 완료율 (80% 등) | CLAUDE.md | Claude가 판단할 때 쓰는 현재 상태 |
| 설계 원칙 (왜 이렇게 하는가) | SOUL.md | 시간이 지나도 낡지 않는 이유 |

## 주의사항

- 내용을 삭제하기 전에 반드시 다른 파일로 이동 또는 링크 삽입 먼저.
- 파일이 존재하지 않으면 새로 만든다. 빈 파일을 만들지 않는다 — 이동할 내용이 없으면 만들지 않는다.
- 스킬 문서(SKILL.md 등) 내부의 중복은 이 스킬의 범위 밖이다. CLAUDE.md·SOUL.md·README.md 세 파일만 대상으로 한다.
